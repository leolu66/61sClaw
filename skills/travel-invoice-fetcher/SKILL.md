# 差旅发票抓取器 (Travel Invoice Fetcher)

## 需求说明（SRS）

### 触发条件
用户说什么话会触发此技能：

| 用户输入 | 含义 |
|---------|------|
| "导入行程单" / "导入行程" | 导入最近1笔FOL行程 |
| "导入3笔行程单" / "导入三个行程" | 导入最近3笔FOL行程 |
| "生成报销单" | 为已有行程目录生成报销单 |
| "生成最近三笔行程单" | 导入3笔FOL行程 + 抓发票 + 生成报销单 |
| "抓取差旅发票" | 手动输入行程描述抓发票 |
| "travel invoice" | 英文触发 |

### 功能描述
**完整流程**：用户输入 → **行程解析(FOL导入)** → 创建目录 → **QQ邮箱抓取** → 整理文件 → **生成报销单**

| 步骤 | 模块 | 说明 |
|------|------|------|
| 1 | 行程解析(FOL导入) | 从FOL财务系统获取"我的申请单"，默认1笔，可指定多笔 |
| 2 | 创建目录 | 在报销凭证目录下创建行程子目录 |
| 3 | QQ邮箱抓取 | 连接QQ邮箱IMAP，搜索发票邮件，下载附件 |
| 4 | 整理文件 | 非PDF文件移到bak/子目录 |
| 5 | 生成报销单 | 扫描发票文件，识别类型，生成Excel报销单 |

**FOL 行程导入**
- 从 FOL 财务系统自动获取"我的申请单"
- 支持指定导入数量（默认1条，可指定多条）
- 自动解析行程信息（出发地、目的地、日期、交通工具）

**报销单生成**
- 扫描行程目录下的发票文件（XML、PDF）
- 智能识别发票类型（高铁票、打车票、机票）
- 自动提取发票金额
- 将发票与行程智能匹配
- 生成标准格式报销单 Excel
- 生成发票匹配报告（已匹配/未匹配）

### 输入/输出
- **输入**: 用户指令（触发词）
  - `导入行程单` → 导入最近1笔FOL行程，抓发票，生成报销单
  - `导入3笔行程单` → 导入最近3笔FOL行程，抓发票，生成报销单
  - `生成报销单` → 为已有行程目录生成报销单
  - `抓取差旅发票：2月8日到3月16日，南京到北京，高铁` → 手动输入行程
- **输出**: 
  - 行程目录（含发票文件）
  - 报销单.xlsx
  - 发票匹配报告.txt

### 依赖条件
- Python 3.7+
- QQ邮箱IMAP访问已开启
- 邮箱凭据存储在 vault 中（key: qq_email）
- fol-login 技能（用于访问 FOL 系统）
- 依赖库: imaplib, email, zipfile, py7zr（可选，用于7z解压）
- **新增依赖**: openpyxl（用于生成 Excel）

### 边界情况
- 邮箱连接失败: 提示检查网络和IMAP设置
- 无匹配邮件: 提示最近7天无发票邮件
- 目录已存在: 覆盖处理
- 日期解析失败: 提示支持的日期格式
- 附件下载失败: 记录错误但继续处理其他邮件
- FOL 导入失败: 提示手动输入行程信息
- **发票识别失败**: 记录到未匹配列表，不影响其他发票处理
- **无发票文件**: 生成空报销单框架，提示用户补充发票

## 使用方法

### FOL 导入（自动抓发票+生成报销单）

| 命令 | 说明 |
|------|------|
| `导入行程单` | 导入最近1笔FOL行程，抓发票，生成报销单 |
| `导入3笔行程单` | 导入最近3笔FOL行程，抓发票，生成报销单 |
| `生成最近三笔行程单` | 同上，导入3笔 + 抓发票 + 生成报销单 |

```bash
# 方式1：使用触发词
python scripts/invoice_fetcher.py fol

# 方式2：指定数量
python scripts/invoice_fetcher.py --fol 3
```

### 手动输入行程

```bash
python scripts/invoice_fetcher.py "2月8日到3月16日，南京到北京，高铁"
```

### 单独生成报销单

```bash
python scripts/expense_report_generator.py "D:\Users\luzhe\报销凭证\南京-北京-0208-0316"
```

- "2月8日" / "3月16日"
- "2025-02-08" / "2025-03-16"
- "02-08" / "03-16"
- "2/8" / "3/16"
- "3月1日-3月5日" / "3月1日到3月5日"

  python invoice_fetcher.py "南京到北京，2月8日到3月16日"

## 报销单生成功能详解

### 发票识别逻辑

1. **扫描目录下所有文件**（包括 zip 解压后的文件）
2. **支持格式**：
   - XML 文件（优先读取元数据）
   - PDF 文件（OCR 识别或文本提取）
3. **发票类型识别**：
   - **高铁票**：标题含"铁路电子客票"、"火车票"、"中国铁路"
   - **打车票**：内容含"旅客运输服务"、"出租车"、"网约车"、"滴滴"、"T3"、"曹操"
   - **机票**：内容含"航空运输电子客票行程单"、"机票"、"航班"、"航空"
4. **智能关联相关票据**：
   - 当文件名没有明显关联时（如 `invoice.pdf` 和 `trip.pdf`）
   - 通过比较**金额**和**文件时间**来判断是否为同一路段的相关票据
   - 关联条件：**金额相同**（允许0.01元误差）+ **文件时间相近**（5分钟内）
   - 关联后的票据会一起显示在报销单的"发票文件"列中

### 发票与行程匹配逻辑

**核心原则：根据实际发票如实填报，不强制匹配预设路段**

1. **发票驱动填报**
   - 用户实际有哪些发票，就填报哪些行程
   - 不强制要求每个预设路段都必须有发票
   - 如果只有去程发票，只报去程；有往返发票，报往返

2. **日期优先匹配**
   - 从发票文件名提取日期前缀（如 `260315_34.00_xxx.pdf` 中的 260315）
   - 将行程日期转换为 YYMMDD 格式进行匹配
   - 日期匹配的发票优先分配给对应日期的路段

3. **类型匹配**
   - 高铁/火车路段 → 匹配火车票
   - 打车/出租车/网约车路段 → 匹配打车票
   - 机票路段 → 匹配机票

4. **城市匹配**
   - 通过发票文件名中的城市关键词判断归属（如"南京"、"北京"）
   - 出发城市路段优先匹配含出发城市关键词的发票
   - 目的城市路段优先匹配含目的城市关键词的发票
   - 火车票不检查城市（因为票面已包含路线信息）

5. **关联票据处理**
   - 主发票匹配到路段后，其关联的行程单自动跟随显示
   - 报销单中显示主发票文件名，备注中列出关联票据

6. **未匹配发票处理**
   - 无法匹配到预设路段的发票，单独列出供用户手动处理
   - 用户可根据实际情况调整行程描述或补充发票

### 报销单日期规则

**报销单中的出发日期只从关联的行程单 PDF 中提取**：

1. **优先提取"行程起止日期"**
   - 解析行程单中的 "行程起止日期：2026-02-25 至 2026-02-25"
   - 取第一个日期（起始日期）作为出发日期
   - 支持格式：`2026-02-25`、`2026年02月25日`

2. **其次从行程明细表格提取**
   - 如果行程单中没有"行程起止日期"，从行程明细表格第一行提取日期
   - 支持格式：`2026年02月25日`、`02-25`（自动补全年份）

3. **提取不到则留空**
   - 如果无法从行程单提取日期，报销单中的出发日期留空
   - 用户需手动确认并填写正确日期

### 报销单 Excel 模板

根据行程结构生成：
- 一个行程 = 去程 + 回程
- 每段包含 2-4 段路径：
  1. 出发地 → 中转站点（打车）
  2. 中转站点 → 出发长途站点（高铁/飞机）
  3. 到达长途站点 → 到达地中转站点（高铁/飞机）
  4. 到达地中转站点 → 目的地（打车）

**Excel 包含字段**：
| 序号 | 路线 | 出发日期 | 返回日期 | 交通工具 | 金额(元) | 发票匹配 | 发票文件 |
|------|------|----------|----------|----------|----------|----------|----------|

### 输出文件

1. **报销单.xlsx** - 标准格式报销单，可直接用于财务报销
2. **发票匹配报告.txt** - 包含：
   - 扫描时间
   - 发票总数统计
   - 已匹配发票详情
   - 未匹配发票列表

## 注意事项

1. **邮箱配置**: 确保QQ邮箱已开启IMAP服务，并生成授权码
2. **凭据存储**: 邮箱账号和密码/授权码需存储在 vault 中，key为 `qq_email`
3. **日期范围**: 默认搜索最近7天的邮件，如需调整请修改配置
4. **文件覆盖**: 已存在的文件会被覆盖，请提前备份重要文件
5. **网络要求**: 需要能访问 imap.qq.com
6. **FOL 导入**: 需要 fol-login 技能已安装，且能访问 FOL 系统
7. **报销单生成**: 
   - 需要安装 openpyxl: `pip install openpyxl`
   - PDF OCR 需要额外安装: `pip install pdf2image pytesseract`（可选）
   - 建议在抓取发票后自动生成报销单

## 架构设计原则

### 分层架构

`
┌─────────────────────────────────────────────────────────┐
│                    业务编排层                            │
│              (invoice_fetcher.py)                       │
│    流程串联：解析行程 → 抓发票 → 归集 → 生成报表          │
└─────────────────────────────────────────────────────────┘
                           │
           ┌───────────────┴───────────────┐
           ▼                               ▼
    ┌─────────────────────────┐   ┌─────────────────────────┐
    │      业务功能模块        │   │      业务功能模块        │
    │  fol_importer.py       │   │  invoice_aggregator.py  │
    │  (从FOL抓取行程)        │   │  (发票归集/匹配)         │
    └─────────────────────────┘   └─────────────────────────┘
           │                               │
           └───────────────┬───────────────┘
                           ▼
    ┌─────────────────────────────────────────────────────────┐
    │                    基础设施层（外部技能）                │
