# 差旅发票抓取器 (Travel Invoice Fetcher)

## 需求说明（SRS）

### 触发条件
用户说什么话会触发此技能：
- "抓取差旅发票"
- "获取出差发票"
- "下载行程发票"
- "travel invoice"
- "抓取发票"
- "获取行程发票"
- "导入行程" / "fol导入行程"
- "导入最近3条行程"
- "生成报销单" / "创建报销单" / "报销单"

### 功能描述
根据用户描述的差旅行程信息，自动：
1. 解析行程信息（出发地、目的地、日期范围）
2. 创建对应的目录结构
3. 从QQ邮箱抓取包含"发票"关键词的邮件
4. 下载邮件附件到对应目录
5. 自动解压压缩文件
6. **智能生成报销单 Excel**（新增功能）

**新增功能：FOL 行程导入**
- 从 FOL 财务系统自动获取"我的申请单"
- 支持指定导入数量（默认1条，可指定多条）
- 自动解析行程信息（出发地、目的地、日期、交通工具）

**新增功能：报销单生成**
- 扫描行程目录下的发票文件（XML、PDF）
- 智能识别发票类型（高铁票、打车票、机票）
- 自动提取发票金额
- 将发票与行程智能匹配
- 生成标准格式报销单 Excel
- 生成发票匹配报告（已匹配/未匹配）

### 输入/输出
- **输入**: 
  - 用户文本描述的行程信息，支持多段行程
    - 示例1: "2月8日到3月16日，南京到北京，高铁"
    - 示例2: "3月1日-3月5日上海出差，3月10日-3月15日深圳出差，都是飞机"
  - FOL 导入指令
    - 示例1: "--fol" 或 "导入行程"（导入最近1条）
    - 示例2: "--fol 3" 或 "导入最近3条行程"（导入最近3条）
  - 报销单生成指令
    - 示例: "生成报销单 D:\\Users\\luzhe\\报销凭证\\南京-北京-0208-0316"
- **输出**: 
  - 创建的目录路径
  - 下载的发票文件列表
  - 报销单 Excel 文件
  - 发票匹配报告
  - 执行日志

### 依赖条件
- Python 3.7+
- QQ邮箱IMAP访问已开启
- 邮箱凭据存储在 vault 中（key: qq_email）
- fol-login 技能（用于访问 FOL 系统）
- 依赖库: imaplib, email, zipfile, py7zr（可选，用于7z解压）
- **新增依赖**: openpyxl（用于生成 Excel）

### 边界情况
- 邮箱连接失败: 提示检查网络和IMAP设置
- 无匹配邮件: 提示最近7天无发票邮件
- 目录已存在: 覆盖处理
- 日期解析失败: 提示支持的日期格式
- 附件下载失败: 记录错误但继续处理其他邮件
- FOL 导入失败: 提示手动输入行程信息
- **发票识别失败**: 记录到未匹配列表，不影响其他发票处理
- **无发票文件**: 生成空报销单框架，提示用户补充发票

## 使用方法

### 基本用法 - 手动输入行程

直接描述行程：

```
抓取差旅发票：2月8日到3月16日，南京到北京，高铁
```

或：

```
获取出差发票：3月1日-3月5日上海出差，3月10日-3月15日深圳出差，都是飞机
```

### FOL 导入 - 自动获取行程

**导入最近1条申请单：**
```bash
python scripts/invoice_fetcher.py --fol
```
或
```bash
python scripts/invoice_fetcher.py fol
```

**导入最近3条申请单：**
```bash
python scripts/invoice_fetcher.py --fol 3
```
或
```bash
python scripts/invoice_fetcher.py fol 3
```

### 生成报销单

**命令行方式：**
```bash
python scripts/expense_report_generator.py "D:\Users\luzhe\报销凭证\南京-北京-0208-0316"
```

**自动触发：**
抓取发票后自动调用报销单生成功能

### 支持的日期格式

- "2月8日" / "3月16日"
- "2025-02-08" / "2025-03-16"
- "02-08" / "03-16"
- "2/8" / "3/16"
- "3月1日-3月5日" / "3月1日到3月5日"

### 目录结构

创建在 `D:\Users\luzhe\报销凭证` 下：

```
报销凭证/
├── 南京-北京-0208-0316/
│   ├── 发票1.pdf
│   ├── 发票2.pdf
│   ├── 报销单.xlsx          # 生成的报销单
│   └── 发票匹配报告.txt      # 匹配报告
└── 上海-深圳-0301-0315/
    ├── 发票1.pdf
    ├── 报销单.xlsx
    └── 发票匹配报告.txt
```

## 报销单生成功能详解

### 发票识别逻辑

1. **扫描目录下所有文件**（包括 zip 解压后的文件）
2. **支持格式**：
   - XML 文件（优先读取元数据）
   - PDF 文件（OCR 识别或文本提取）
3. **发票类型识别**：
   - **高铁票**：标题含"铁路电子客票"、"火车票"、"中国铁路"
   - **打车票**：内容含"旅客运输服务"、"出租车"、"网约车"、"滴滴"、"T3"、"曹操"
   - **机票**：内容含"航空运输电子客票行程单"、"机票"、"航班"、"航空"

### 报销单 Excel 模板

根据行程结构生成：
- 一个行程 = 去程 + 回程
- 每段包含 2-4 段路径：
  1. 出发地 → 中转站点（打车）
  2. 中转站点 → 出发长途站点（高铁/飞机）
  3. 到达长途站点 → 到达地中转站点（高铁/飞机）
  4. 到达地中转站点 → 目的地（打车）

**Excel 包含字段**：
| 序号 | 路线 | 出发日期 | 返回日期 | 交通工具 | 金额(元) | 发票匹配 | 发票文件 |
|------|------|----------|----------|----------|----------|----------|----------|

### 输出文件

1. **报销单.xlsx** - 标准格式报销单，可直接用于财务报销
2. **发票匹配报告.txt** - 包含：
   - 扫描时间
   - 发票总数统计
   - 已匹配发票详情
   - 未匹配发票列表

## 注意事项

1. **邮箱配置**: 确保QQ邮箱已开启IMAP服务，并生成授权码
2. **凭据存储**: 邮箱账号和密码/授权码需存储在 vault 中，key为 `qq_email`
3. **日期范围**: 默认搜索最近7天的邮件，如需调整请修改配置
4. **文件覆盖**: 已存在的文件会被覆盖，请提前备份重要文件
5. **网络要求**: 需要能访问 imap.qq.com
6. **FOL 导入**: 需要 fol-login 技能已安装，且能访问 FOL 系统
7. **报销单生成**: 
   - 需要安装 openpyxl: `pip install openpyxl`
   - PDF OCR 需要额外安装: `pip install pdf2image pytesseract`（可选）
   - 建议在抓取发票后自动生成报销单
