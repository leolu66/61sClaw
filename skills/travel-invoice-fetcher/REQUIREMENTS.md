# 差旅发票抓取器需求文档

## 一、需求背景

用户（六一）经常需要处理差旅报销，流程繁琐：
1. 从 FOL 财务系统复制行程信息
2. 手动创建文件夹整理发票
3. 从 QQ 邮箱翻找发票邮件
4. 手动填写报销单

**痛点**：重复劳动、容易遗漏、效率低

---

## 二、核心需求

| 需求 | 说明 |
|------|------|
| 自动导入行程 | 从 FOL 财务系统获取"我的申请单" |
| 自动抓取发票 | 连接 QQ 邮箱，按日期/城市搜索下载 |
| 智能识别发票 | 自动识别高铁票、打车票、机票类型 |
| 自动生成报销单 | 扫描发票生成标准 Excel 报销单 |
| 智能关联票据 | 将发票和行程单自动关联 |

---

## 四、模块详解

### 4.1 行程解析模块 (trip_parser.py)

**功能**：从自然语言提取结构化行程

**支持的输入格式**：
```
"2月8日到3月16日，南京到北京，高铁"
"3月1日-3月5日上海出差，3月10日-3月15日深圳出差，都是飞机"
"2025-02-08到2025-03-16，南京到北京"
```

**输出**：`TripSegment` 对象
- departure_city: 出发城市
- destination_city: 目的城市  
- start_date: 开始日期
- end_date: 结束日期
- transport_mode: 交通方式

**目录命名规则**：`出发地-目的地-MMDD-MMDD`
例：`南京-北京-0208-0316`

---

### 4.2 FOL 导入模块 (fol_importer.py)

**功能**：从 FOL 财务系统导入申请单

**实现方式**：
- 目前为**半自动**：用户复制 FOL 列表内容粘贴
- 解析表格行数据，提取单号、项目、描述、时间
- 根据描述推断出发地、目的地、交通工具

**数据结构**：
```python
@dataclass
class FOLApplication:
    form_no: str      # 单号 (如 1-SQ12026002686)
    form_name: str    # 表单名称
    project: str      # 项目名称
    description: str  # 描述
    submit_time: str  # 提交时间
    status: str       # 单据状态
```

---

### 4.3 邮件抓取模块 (email_fetcher.py)

**功能**：通过 IMAP 连接 QQ 邮箱抓取发票邮件

**核心逻辑**：
```python
# 1. 连接邮箱
mail = imaplib.IMAP4_SSL("imap.qq.com", 993)
mail.login(email, password)

# 2. 搜索邮件（最近 N 天）
search_criteria = f'(SINCE "{since_date}")'

# 3. 筛选主题含"发票"的邮件
# 4. 排除通信运营商发票（移动/联通/电信）
# 5. 下载附件，自动解压 zip/rar/7z
```

**特殊处理**：
- 排除通信运营商发票（话费发票不需要报销）
- 限制处理最新 50 封邮件（性能考虑）
- 自动解压压缩文件

---

### 4.4 报销单生成模块 (expense_report_generator.py)

这是最复杂的模块，包含三个核心功能：

#### 4.4.1 发票识别 (InvoiceRecognizer)

**支持格式**：
- XML 文件（电子发票元数据）
- PDF 文件（OCR 识别或文本提取）

**识别逻辑**：
```python
# 优先从文件名提取金额（最准确）
# 文件名格式: 260315_34.00_北京滴滴.pdf

# 识别发票类型：
if '铁路电子客票' in content or '火车票' in content:
    type = TRAIN
elif '旅客运输服务' in content or '滴滴' in content:
    type = TAXI
elif '航空' in content or '航班' in content:
    type = FLIGHT
```

#### 4.4.2 智能关联相关票据

**问题**：一张打车行程通常有两张 PDF：
- 发票（用于报销）
- 行程单（显示起点终点）

**关联规则**：
```python
# 规则1：相同日期前缀 + 相同金额
# 260315_34.00_北京滴滴.pdf ↔ 260315_34.00_行程单.pdf

# 规则2：文件名匹配
# invoice.pdf ↔ trip.pdf（金额相同）
```

#### 4.4.3 发票与行程匹配

**核心原则**：**发票驱动填报**，不强制匹配预设路段

**匹配策略**：
1. **日期优先**：从文件名提取日期前缀（如 260315），匹配对应日期的路段
2. **类型匹配**：高铁路段 → 火车票，打车路段 → 打车票
3. **城市匹配**：通过文件名城市关键词判断归属
4. **关联票据跟随**：主发票匹配后，关联的行程单自动跟随

**匹配示例**：
```
行程: 南京 → 北京, 2026-02-25 至 2026-02-27
路段:
  - 去程-1: 家 → 南京南站 (打车)
  - 去程-2: 南京南站 → 北京南站 (高铁)
  - 去程-3: 北京南站 → 宿舍 (打车)

发票:
  - 260225_45.00_南京滴滴.pdf → 匹配去程-1
  - 260225_476.50_铁路电子客票.pdf → 匹配去程-2
  - 260225_34.00_北京滴滴.pdf → 匹配去程-3
```

#### 4.4.4 报销单 Excel 生成

**Excel 字段**：
| 序号 | 路线 | 出发日期 | 交通工具 | 金额(元) | 发票匹配 | 发票文件 |

**特殊处理**：
- 出发日期**只从行程单 PDF 中提取**（最准确）
- 关联票据的文件名一起显示在"发票文件"列
- 未匹配发票单独列出在匹配报告中

## 五、使用流程

### 5.1 完整流程（FOL导入）
```
用户: "导入3笔行程单"
  ↓
读取 FOL 申请单列表（用户粘贴）
  ↓
解析最近3条申请单
  ↓
为每条行程创建目录
  ↓
连接 QQ 邮箱，搜索发票邮件
  ↓
下载附件到对应目录
  ↓
整理文件（非PDF移入bak/）
  ↓
扫描发票，识别类型和金额
  ↓
智能关联相关票据
  ↓
按路段匹配发票
  ↓
生成报销单 Excel
  ↓
生成发票匹配报告
```

### 5.2 输出文件
```
报销凭证/
├── 南京-北京-0208-0316/
│   ├── 260225_45.00_南京滴滴.pdf
│   ├── 260225_476.50_铁路电子客票.pdf
│   ├── 260225_34.00_北京滴滴.pdf
│   ├── 报销单_143052.xlsx      # 生成的报销单
│   ├── 发票匹配报告.txt         # 匹配详情
│   └── bak/                     # 非PDF文件
│       └── 行程单.pdf
```

## 六、技术亮点
