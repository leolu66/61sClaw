# 创建日志迁移定时任务指南

由于需要管理员权限，建议手动创建定时任务。

## 方法 1：使用任务计划程序（推荐）

### 步骤：

1. **打开任务计划程序**
   - 按 `Win + R` 键
   - 输入 `taskschd.msc`
   - 按回车

2. **创建基本任务**
   - 在右侧点击"创建基本任务"
   - 或右键"任务计划程序库" → "创建基本任务"

3. **设置常规选项**
   - **名称**：日志自动归档
   - **描述**：每天晚上 23:20 自动迁移超过 7 天的工作日志
   - **安全选项**：
     - ☑️ 不管用户是否登录都要运行
     - ☑️ 不存储密码

4. **设置触发器**
   - 点击"触发器"选项卡
   - 点击"新建..."
   - 设置：
     - 开始任务：选择"每天"
     - 开始：23:20:00
   - 点击"确定"

5. **设置操作**
   - 点击"操作"选项卡
   - 点击"新建..."
   - 设置：
     - 操作：启动程序
     - 程序或脚本：`C:\Users\luzhe\.openclaw\workspace-main\skills\log-migrator\scripts\migrate_logs.bat`
     - 起始于（可选）：`C:\Users\luzhe\.openclaw\workspace-main\skills\log-migrator\scripts\`
   - 点击"确定"

6. **设置条件（可选）**
   - 点击"条件"选项卡
   - 取消勾选"只有在计算机使用交流电源时才启动此任务"
   - 这样确保即使用电池也会运行

7. **设置设置（可选）**
   - 点击"设置"选项卡
   - ☑️ 如果任务失败，按以下频率重新启动
     - 每：5 分钟
     - 尝试次数：3 次
   - ☑️ 如果任务运行时间超过以下时间，停止任务
     - 持续时间：1 小时

8. **完成**
   - 点击"确定"保存任务

## 方法 2：使用命令行（需要管理员权限）

### 以管理员身份运行 PowerShell：

```powershell
# 1. 右键点击 PowerShell 图标
# 2. 选择"以管理员身份运行"

# 3. 执行以下命令
schtasks /Create /TN "日志自动归档" `
  /TR "C:\Users\luzhe\.openclaw\workspace-main\skills\log-migrator\scripts\migrate_logs.bat" `
  /SC DAILY `
  /ST 23:20 `
  /RU "SYSTEM" `
  /RL HIGHEST `
  /F
```

## 故障排除

### 问题：任务未执行

**可能原因**：
- 系统休眠或关机
- 账户权限不足
- 触发时间设置错误

**解决方法**：
- 检查任务计划程序中的任务历史记录
- 确保勾选"不管用户是否登录都要运行"
- 检查触发器设置是否正确

### 问题：脚本执行失败

**可能原因**：
- Python 路径未添加到系统 PATH
- 脚本文件权限问题
- 依赖库未安装

**解决方法**：
- 手动运行脚本测试：`migrate_logs.bat`
- 检查 Python 是否可用：`python --version`
- 查看迁移日志：`logs/errors/log-migration-*.md`

## 管理任务

### 启用任务：
```powershell
schtasks /Change /TN "日志自动归档" /Enable
```

### 禁用任务：
```powershell
schtasks /Change /TN "日志自动归档" /Disable
```

### 删除任务：
```powershell
schtasks /Delete /TN "日志自动归档" /F
```
