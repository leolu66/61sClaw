# model-recommender - 模型推荐技能

根据用户需求推荐最适合的模型，支持按场景、价格、能力等维度推荐。

## 触发条件

当用户询问以下内容时触发：
- "推荐模型"
- "用什么模型"
- "哪个模型便宜"
- "编程用什么模型"
- "支持图像的模型"
- "长文档用什么模型"
- 以及其他模型选择相关的问题

## 功能

1. **显示完整模型价格排行榜** - 按价格排序展示所有可用模型
2. **按场景推荐** - 根据用户的使用场景推荐最合适的模型
3. **编程模型对比** - 专门针对编程任务对比各模型
4. **性价比分析** - 综合价格和能力的推荐

## 数据源

- `models-capabilities.json` - 11 个模型的完整价格、上下文、标签数据
- 包含推荐矩阵和编程模型对比

## 输出格式

### 默认输出（当用户说"推荐模型"）

1. 模型价格排行榜（前 3 名带奖牌🥇🥈🥉）
2. 场景推荐矩阵表格
3. 当前用户默认模型建议

### 特定场景输出

根据用户场景返回对应的首选/备选/高端推荐

## 价格说明

- 单位：元 / 1M tokens
- 缓存价格：命中缓存时的优惠价格
- 上下文：模型支持的最大 token 数

## 推荐原则

### 够用就行

> 在满足需求的前提下，追求最低成本，不盲目追求最新最酷的技术。

**核心要点：**
- **需求优先** - 先明确实际需求，再选择模型
- **成本意识** - 在符合要求的基础上选最便宜的
- **拒绝过度** - 不追求"最好"，只追求"最合适"
- **务实选择** - 适合自己的才是最好的

- 价格可能随时间变化，定期更新 JSON 文件
- 推荐基于性价比，不追求绝对最好
- 考虑用户的具体需求和预算约束
