## 修复 skill-syncer SSL 握手超时

**问题**: Python urllib 在 Windows 上请求 raw.githubusercontent.com 时 SSL 握手超时

**解决方案**:
- 修改 `github_client.py`，优先使用 curl 发送 HTTP 请求
- curl 失败时回退到 urllib，保持兼容性
- 参数: `-s` 静默, `-L` 跟随重定向, `--max-time 30` 超时

**经验**:
- Windows 上 urllib 的 SSL 实现不稳定，curl 更可靠
- 网络请求优先使用系统级工具
- 保留回退方案确保兼容性

## 多智能体协调器心跳机制优化

**任务**: 优化 multi-agent-coordinator 技能的定时扫描机制

**成果**:
1. 独立定时扫描程序（MasterScanner/WorkerScanner）
2. 统一配置文件 `coordinator.json`（支持热重载）
3. CLI 工具（start/stop/status/reload）
4. 状态文件通信机制（PID + status 文件）
5. 技能主入口 `index.js`

**关键设计**:
- 扫描间隔 30 秒，心跳 5 分钟
- 状态文件每 30 秒更新，技能通过读取状态文件确认执行结果
- 配置分层：统一入口 + 本地覆盖 + 运行时状态

**经验**:
- 后台进程通过文件状态通信比直接返回更可靠
- 配置集中管理便于维护和热更新
- 实现顺序：配置 → 加载类 → CLI → 扫描器 → 技能入口
