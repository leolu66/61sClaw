# 2026-03-02 工作日志

## 任务：修复 API 余额查询 Chrome 多开问题

### 问题描述
- 查余额时不停 kill Chrome 进程，然后又重启
- Chrome 会打开两个窗口（一个空白，一个 Dashboard）
- 还会影响到用户正在使用的 Chrome

### 根因分析
1. 使用 `subprocess.Popen` 启动 Chrome + `connect_over_cdp` 连接，导致两个 Chrome 实例
2. `taskkill /F /IM chrome.exe` 会杀掉所有 Chrome 进程
3. 端口 9222 检测不可靠（显示开着但实际连接失败）

### 解决方案
1. 使用 Playwright 的 `launch_persistent_context` 替代 subprocess 启动 Chrome
2. 只启动一个 Chrome 窗口，复用登录状态
3. 移除所有手动 kill Chrome 的逻辑
4. 添加连接失败自动重启的重试机制

### 代码改动
- `api_balance_checker.py`: 重构 Chrome 启动和连接逻辑
- 删除 15 个调试/过程文件，只保留核心文件
- 代码从 1200+ 行精简到 500+ 行

### 测试结果
- ✅ 只打开一个 Chrome 窗口
- ✅ 登录状态正确复用
- ✅ 连接失败时自动重启
- ✅ 不影响用户正在使用的 Chrome
