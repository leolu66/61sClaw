## 学到的新知识

- session 文件容易被污染（工具输出、脚本错误等），难以清洗
- 模型本身是最好的内容提取器，因为它能理解上下文
- 记忆管理系统：工作日志（logs/daily/）= 详细记录，每日记忆（memory/）= 提炼总结
- 推送历史时间比较：批量推送时更高效，只推送更新的技能

## 错误记录和反思

- 问题：auto_logger.py 从 session 文件提取内容时，混入了大量系统元数据
- 教训：不应该从被污染的 session 文件提取内容，应该让模型直接生成
- 问题：在 openclaw.json bindings 中添加 accountId 字段导致启动失败
- 教训：bindings 不支持 accountId 字段，需要查文档确认正确方案
- 问题：心跳告警发到了小飞的 OpenClaw 机器人，而不是六一的 oc_xbot
- 修复：交换 channels.feishu.accounts.default 和 oc_xbot 的值

## 开发经验总结

- 遇到内容提取困难时，考虑换个思路：让模型直接生成，而不是从文件提取
- 模型直接生成的好处：干净、准确、能理解上下文
- 批量操作应该智能跳过：无更新的技能不需要重复推送
- 结果展示应该有针对性：只显示需要用户关注的内容
