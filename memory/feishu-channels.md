## 跨应用通信限制

**重要：不同飞书应用之间无法直接通信！**

### 测试结果

| 发送方 | 接收方 | 会话ID | 结果 | 错误 |
|--------|--------|--------|------|------|
| OC_XBot | 小飞P2P Chat | `oc_438958a084d87cf6e20cf37647f6593c` | ❌ 失败 | `Bot/User can NOT be out of the chat` (230002) |
| OC_XBot | 小飞P2P | `ou_6296b58fee28100a7b01c0b8d0fb631f` | ❌ 失败 | `open_id cross app` (99992361) |

### 结论

1. **不同应用的会话ID不能混用**
   - `ou_6296b58fee28100a7b01c0b8d0fb631f` 仅在小飞应用中有效
   - OC_XBot 应用无法使用，报 cross app 错误

2. **机器人不能给非本应用的用户/群聊发消息**
   - 即使知道会话ID，也会报 `out of the chat` 错误

3. **可行的跨应用通信方式**
   - ✅ 把两个机器人拉进同一个群聊（如果支持）
   - ✅ 用户作为中转（用户同时与两个机器人对话）
   - ✅ 通过外部系统（如数据库、消息队列）中转

## 飞书 @ 功能限制

**无法通过 API 发送真正的 @ 消息**

### 当前能力

| 操作 | 支持 | 说明 |
|------|------|------|
| 纯文本包含 "@用户名" | ✅ | 仅显示为普通文本，不会触发提醒 |
| 真正的 @ 提醒 | ❌ | 需要特殊的消息格式，当前不支持 |

### 为什么不能真正 @

飞书的 @ 功能需要特定的消息格式：

```json
{
  "content": {
    "at": {
      "atUser": {
        "userId": "ou_xxx"
      }
    },
    "text": "消息内容"
  }
}
```

**限制原因：**
1. `openclaw message send` 命令不支持飞书的富文本 `at` 格式
2. 需要知道被 @ 用户在当前群里的 user_id
3. 底层需要支持飞书的消息卡片格式

### 替代方案

1. **用户手动 @** - 你在群里手动 @OpenClawd，我可以看到消息并回复
2. **私聊通知** - 直接给 OpenClawd 私聊发消息（如果知道其会话ID）
3. **群公告** - 发送重要消息到群里，所有人都能看到

## 常见问题

### open_id cross app 错误

**错误信息**：`code: 99992361, msg: 'open_id cross app'`

**原因**：使用的 open_id 不属于当前飞书应用

**解决**：
1. 确认消息来源的应用
2. 使用该应用对应的 open_id
3. 或改用群聊 chat_id 发送（chat_id 通常跨应用可用）
