## 决策原则

### 让子弹飞一会

> 任何决策或选择，如果当前情况还没有出现严重问题或冲突，而你也没有想清楚，可以暂缓决策，先并行或同步再运行一段时间，让子弹飞一会。

**适用场景：**
- 技术选型对比（框架A vs 框架B）
- 架构方案选择
- 是否进行大规模重构
- 新功能是否值得投入开发
- 多方案并行验证效果

**为什么有效：**
- 避免因信息不足而做出次优选择
- 减少过早决策导致的方向错误
- 给系统/方案更多时间展现真实表现
- 降低试错成本

**执行方式：**
1. 明确暂缓决策的期限（如：1周、1个月）
2. 设定观察指标和对比维度
3. 并行运行多个方案，收集数据
4. 期限到达后，基于数据做出决策

## 工程实践原则

### 分层设计原则

> 下层能力要简单、可靠、独立，复杂的功能在上层再组装，增加灵活性。

**核心要点：**
- **底层简单可靠** - 每个底层模块只做一件事，做到极致稳定
- **上层组装灵活** - 复杂功能通过组合底层模块实现
- **保持独立性** - 底层不依赖上层，便于替换和复用
- **分层解耦** - 改变一层不影响其他层

### 领域模型构建原则

> 要在交互中，逐步构建总体的视图和模型，在这个模型基础上去设计程序会更灵活。

**核心要点：**
- **先有模型，再写代码** - 在实践中形成对领域的认知，构建模型
- **总体视图先行** - 先画大图，了解全貌，再切分模块
- **路径/分类/策略** - 每个功能形成清晰的路径、分类、策略
- **便于迭代** - 模型清晰后，后期维护和扩展都更方便

**示例（几何万国游戏）：**
- 构建游戏总体视图（资源、任务、地图等）
- 分层模型（底层操作 → 中层策略 → 高层决策）
- 每个功能路径：领取奖励 → 检查条件 → 执行领取 → 记录状态
- 分类清晰：日常任务 / 随机事件 / 长期目标
- 策略分明：什么时候做什么最优化

**记录时间：** 2026-02-21

### 复杂任务分解原则

> 面对复杂任务时，先进行任务分解，将大任务拆分为相对独立的子功能。

**执行步骤：**
1. **识别核心功能** - 先关注核心功能，再完善细节和辅助功能
2. **模块分解** - 保持每个模块的高内聚、低耦合
3. **简单原型先行** - 先从简单原型和技术验证开始，实现关键技术点的0→1突破
4. **逐步扩展** - 核心功能验证后再考虑1→N的拓展

**示例：**
- 微信公众号文章获取工具分解为：列表获取 → 内容读取 → 文件保存 → 报告生成
- 每个模块独立，通过接口交互

**记录时间：** 2026-02-18

### 外部约束关注原则

> 实现功能时要关注外部资源的约束和限制，避免被禁止或限制。

**反爬策略：**
1. **随机延迟** - 请求间添加随机延迟，避免固定频率
2. **随机顺序** - 处理顺序随机化，避免可预测模式
3. **控制并发** - 避免高密度、大批量并发请求
4. **模拟真实行为** - 使用浏览器模拟、随机User-Agent等

**示例：**
- 微信公众号文章读取：2-5秒随机延迟、随机获取顺序、单线程顺序处理

**记录时间：** 2026-02-18

---

### 可维护性设计原则

> 把可变和不变的内容隔离开来，避免每次修改都要大面积改程序。

**实践方法：**
1. **配置外置** - 将可变参数（公众号列表、路径、阈值等）放到外部配置文件
2. **环境变量** - 敏感信息（密钥、Token等）通过环境变量传入
3. **命名模板** - 支持可配置的命名格式，如 `{account}_{title}.md`
4. **参数化设计** - 延迟时间、超时时间、数量限制等都做成可配置参数

**示例：**
- `config.json` 管理公众号列表，添加新号无需改代码
- 环境变量 `WECHAT_COOKIE` 传入敏感信息

### 飞连VPN失效问题
- **原因**：系统设置了 HTTP_PROXY/HTTPS_PROXY 环境变量会导致飞连 VPN 失效
- **症状**：飞连报错 "出现未知错误，请按提示方法解决或联系企业管理员。 (21110014) 飞连服务被拦截"
- **排查**：如果其他机器访问飞连正常，但本机突然失效，检查系统环境变量是否有 PROXY 相关配置
- **解决**：删除 HTTP_PROXY、HTTPS_PROXY、ALL_PROXY 等代理环境变量
- **记录时间：** 2026-02-23

## 错误学习方法论

### 错误是最好的老师

> 每次遇到错误时，需要去反思过程，建立错误日志，定期从历史错误中检索和学习。

**核心要点：**
- **错误即学习** - 每次错误都是成长的机会，不要逃避或忽略
- **过程反思** - 不仅修复错误，还要理解为什么出错
- **错误日志** - 记录错误类型、原因、解决方案，形成知识库
- **定期回顾** - 从历史错误中检索，避免重复犯错
- **举一反三** - 一个错误的教训可以应用到类似场景

**执行方式：**
1. 遇到错误时，先记录错误现象和上下文
2. 分析根本原因，不只是表象
3. 总结解决方案和预防措施
4. 定期回顾错误日志，强化记忆
5. 将错误经验应用到新任务中
