---

## 2. 两种通信模式

### 模式A：在家（局域网直连）

- 两台机器在同一 WiFi 下
- 通过 HTTP REST API 直接通信
- 实时性强，延迟低

### 模式B：出门（NAS 中转）

- 笔记本不在局域网内
- 通过 Z 盘文件队列中转消息
- 非实时（定时轮询）

---

## 3. 消息格式

统一使用 JSON 格式：

```json
{
  "from": "小天才",
  "to": "小白",
  "type": "task|response|heartbeat",
  "content": {
    "action": "执行xxx",
    "params": {}
  },
  "timestamp": "2026-02-25T00:42:00Z",
  "id": "uuid"
}
```

## 4. 实现步骤

### Phase 1：基础通信层

1. **消息队列模块** (`msg_queue.py`)
   - 发送消息到指定收件箱
   - 拉取未读消息
   - 消息签收（避免重复消费）

2. **在家模式检测** (`network_detect.py`)
   - 检测是否在局域网（同一 WiFi）
   - 自动切换通信模式

### Phase 2：API 服务

3. **HTTP 服务** (`server.py`)
   - 监听端口 8765
   - 提供 `/send` `/poll` `/ack` 接口
   - 暴露给局域网调用

### Phase 3：集成

4. **任务执行器** (`executor.py`)
   - 接收任务消息
   - 执行具体操作
   - 返回执行结果

5. **任务调度** (`scheduler.py`)
   - 定时检查收件箱
   - 在家模式优先直连
