# 日志目录

本目录用于记录工作过程中的各种日志信息，便于回顾和总结。

## 目录结构

```
logs/
├── daily/     # 每日会话总结
│   └── YYYY-MM-DD-NNN-简短概述.md
├── tasks/     # 任务执行日志
│   └── 任务名-时间戳.md
├── errors/    # 错误反思记录
│   └── 错误类型-日期.md
└── summary/   # 阶段性总结
    ├── YYYY-MM-周报.md
    └── YYYY-QN-总结.md
```

## 各子目录说明

### daily/ - 每日会话总结

记录每次会话的工作内容、问题、解决方案。

**命名格式**：`YYYY-MM-DD-NNN-简短概述.md`

**示例**：
- `2026-02-28-001-修复技能路径+账单分析+天气配置.md`
- `2026-02-28-002-添加密码箱账号.md`

**内容模板**：
```markdown
# 工作日志 #NNN

## 基本信息
- 日期：YYYY-MM-DD
- 开始时间：HH:MM
- 结束时间：HH:MM

## 工作内容概述

简要描述本次会话的主要工作内容。

## 完成的任务
- 任务1：描述
- 任务2：描述

## 遇到的问题与解决方案
| 问题描述 | 解决方案 | 结果 |
```

---

### tasks/ - 任务执行日志

记录具体任务的执行过程、中间状态。

**命名格式**：`任务名-YYYY-MM-DD-HHMM.md`

**示例**：
- `sync-github-20260227-1430.md`
- `fix-billing-20260228-1000.md`

---

### errors/ - 错误反思记录

记录错误现象、原因、解决方案，用于学习和避免重复犯错。

**命名格式**：`错误类型-YYYY-MM-DD.md`

**示例**：
- `path-escape-error-20260228.md`
- `api-limit-exceeded-20260227.md`

**内容模板**：
```markdown
# 错误类型 - 日期

## 错误现象
描述错误的具体表现

## 错误原因
分析导致错误的根本原因

## 解决方案
如何修复这个问题

## 经验教训
从这次错误中学到什么，如何避免再次发生
```

---

### summary/ - 阶段性总结

定期总结（周报、月报、季度回顾）。

**命名格式**：
- 周报：`YYYY-MM-WN-周报.md`
- 月报：`YYYY-MM-月报.md`
- 季报：`YYYY-QN-总结.md`

**示例**：
- `2026-02-周报.md`
- `2026-Q1-工作总结.md`

---

## 注意事项

1. **敏感信息过滤**：日志中避免记录密码、API Key 等敏感信息
2. **及时归档**：定期清理或归档旧的日志文件
3. **统一格式**：使用 Markdown 格式，便于阅读和搜索
4. **版本控制**：daily/ 下的日志会随代码一起提交到 GitHub
   - 敏感日志添加到 `.gitignore`
   - 定期清理或归档到外部目录
