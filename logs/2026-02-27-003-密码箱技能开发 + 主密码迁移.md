### 2. Vault 密码箱技能开发

**需求**: 安全存储和查询各平台的账号、密码、API Key 等敏感信息

**实现方案**:
- **存储**: 单 JSON 文件 (`~/.openclaw/vault/credentials.json`)
- **加密**: AES-256-GCM 加密敏感字段
- **主密码**: `XXXXXX`（用于派生加密密钥）
- **预设平台**: 10 个主流平台模板

**核心功能**:
| 命令 | 功能 | 输出 |
|------|------|------|
| `vault query <平台>` | 查询凭据 | 纯明文值 |
| `vault read <平台>` | 查看完整信息 | 详细信息列表 |
| `vault copy <平台>` | 复制凭据 | 纯明文值 |
| `vault list` | 列出所有平台 | 分类列表 |
| `vault save <平台> k=v` | 保存/更新凭据 | ✅ 提示 |

**迭代优化**:
1. 简化查询输出 - 只显示关键信息（身份证号、密码），无多余内容
2. 添加 read 命令 - 显示完整明文供阅读
3. 改为主动复制 - 不自动复制到剪贴板，显示纯文本
4. 渠道区分 - 飞书渠道不提供 copy 功能

### 4. 主密码安全迁移

**问题**: 主密码硬编码在代码中，存在安全风险

**解决方案**: 迁移到 Windows 凭据管理器

**实现**:
- 凭据名称：`OpenClawVaultMasterPassword`
- 修改 `vault.py` 从凭据管理器读取主密码
- 支持环境变量降级和硬编码降级

**安全提升**: 代码泄露不再暴露主密码
