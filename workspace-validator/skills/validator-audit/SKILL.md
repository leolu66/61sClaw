# validator-audit

## 描述

验证审核Agent的核心技能 - 系统化检查所有Agent技能规范性和GitHub仓库合规性。

## 触发方式

- "验证审核"
- "检查技能规范"
- "审计GitHub"
- "隐私安全检查"
- "合规检查"

## 功能

### 1. 技能规范检查

检查 `workspace-main/skills/` 和 `workspace-entertainment/skills/` 下的所有技能：

| 检查项 | 标准 | 检查方法 |
|--------|------|----------|
| SKILL.md 存在 | 必须存在 | 文件系统检查 |
| 触发词 | 必须明确列出 | 解析 SKILL.md |
| 描述清晰 | 功能描述完整 | 内容审查 |
| 相对路径 | 脚本内使用相对路径 | 正则匹配 |
| 配置分离 | 敏感配置放 config.json | 文件检查 |

### 2. 隐私安全检查

扫描敏感信息：

| 类型 | 匹配模式 | 严重程度 |
|------|----------|----------|
| 密码 | `password[=:]\S+` | 🔴 高 |
| API Key | `api_key[=:]\S+` | 🔴 高 |
| Token | `token[=:]\S+` | 🔴 高 |
| 手机号 | `1[3-9]\d{9}` | 🟡 中 |
| 邮箱 | `\w+@\w+\.\w+` | 🟡 中 |
| 身份证号 | `\d{17}[\dXx]` | 🔴 高 |

### 3. GitHub 合规检查

检查仓库上传内容：

| 检查项 | 要求 | 检查位置 |
|--------|------|----------|
| .gitignore | 必须存在且配置正确 | 根目录 |
| node_modules | 必须被忽略 | .gitignore |
| .git 目录 | 必须被忽略 | .gitignore |
| 配置文件 | 敏感配置不上传 | 全仓库 |
| 目录结构 | 符合 workspace 规范 | 根目录 |

## 使用方法

### 全面审计

```
用户：执行全面验证审核
Agent：
1. 扫描 workspace-main/skills/ 所有技能
2. 扫描 workspace-entertainment/skills/ 所有技能
3. 检查 GitHub 仓库合规性
4. 生成审计报告
```

### 定向检查

```
用户：检查 [技能名] 的规范性
Agent：
1. 定位技能目录
2. 检查 SKILL.md 格式
3. 检查脚本文件规范
4. 检查隐私安全
5. 输出检查结果
```
