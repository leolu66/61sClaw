## 学到的新知识

- 仓库重构时，若本地已是目标结构，直接用 `git add` 即可，无需 `git mv`
- `git rm` 可以同时删除文件并标记变更，比手动删除更高效
- 分多次小提交比单个大提交更安全，便于问题定位和回滚

## 错误记录和反思

- 初次操作时用 `git add -A` 差点把 node_modules 也提交进去，以后要注意精确指定路径
- 根目录重复文件第一次没清理干净，需要二次检查确认

## 开发经验总结

- 重构前先确认本地和远程的对应关系，避免误操作
- 每次提交只做一件事，提交信息要清晰描述变更内容
- 文档要随代码同步更新，README 的统计数字要及时修正
