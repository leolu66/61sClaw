# 代码统计技能

统计 skills 目录下所有技能的工作成果。

## 触发方式

用户说以下指令时触发：
- "代码统计"
- "工作统计"
- "技能统计"
- "统计技能"

## 输出格式

```markdown
# 📊 Skills 工作统计报告

## 总体概况
- 技能数量: 33 个
- 总文件数: 203 个
- 总代码行数: 16,343 行
- 总文档行数: 6,057 行
- 总大小: 7.0 MB

## GitHub 提交统计（最近7天）
- 提交次数: 100 次
- 活跃天数: 5 天

| 类型 | 增加 | 删除 | 净变化 |
| ------------------ | ------- | ------ | ------ |
| 代码 (.py/.js) | +30,389 | -6,687 | +23,702 |
| 文档 (.md) | +11,082 | -4,661 | +6,421 |

## Top 5 技能（按代码+文档行数）
| 排名 | 技能 | 代码行数 | 文档行数 |
| ---- | ----------------------- | -------- | -------- |
| 1 | jhwg-auto | 2,654 | 589 |
| 2 | wechat-article-fetcher | 2,307 | 632 |
| 3 | multi-agent-coordinator | 1,341 | 499 |
| 4 | game-auto-clicker | 1,537 | 123 |
| 5 | todo-manager | 1,174 | 346 |
```

## 统计规则

1. **技能数量**: 包含 SKILL.md 的目录数量
2. **文件数量**: 所有文件（不含 node_modules、data、__pycache__）
3. **代码行数**: 仅统计 .py, .js 文件的有效代码行数（排除空行和注释）
4. **大小统计**: 所有文件的总大小
5. **GitHub 统计**: 自动从 vault 读取 GitHub Token，获取最近7天的提交数据

## 依赖

- GitHub Token: 自动从 `vault` 技能读取（credentials.json）
