# 验证、审计相关技能清单

本文档整理 OpenClaw 工作区中所有与**验证、审计、检查、合规**相关的技能，供验证审核Agent参考使用。

---

## 🔴 核心验证审计技能

### 1. github-compliance-checker（GitHub合规检查器）
- **路径**: `workspace-main/skills/github-compliance-checker/`
- **触发词**: "检查 GitHub 合规性"、"审核 GitHub"、"检查 github"、"github 合规检查"
- **功能**:
  - 检查 GitHub 仓库上传合规性
  - 防止隐私文件、敏感信息上传到 GitHub
  - 自动检测并修复不合规文件
  - 支持用户确认模式和自动修复模式
- **检查规则**:
  - 🔴 高风险: MEMORY.md、USER.md、logs/、memory/、个人数据
  - 🟡 中风险: TOOLS.md、技能数据目录
  - 🟢 低风险: __pycache__、IDE配置、临时文件
- **核心脚本**: `scripts/compliance_checker.py`、`scripts/fix_gitignore.py`

---

### 2. skill-conflict-detector（技能冲突检测器）
- **路径**: `workspace-main/skills/skill-conflict-detector/`
- **触发词**: "技能冲突检测"、"分析技能相似度"、"检查技能触发冲突"、"技能分组"
- **功能**:
  - 扫描所有技能的元数据
  - 检测触发词冲突（重复、相似、模糊匹配）
  - 分析功能相似度，自动分组
  - 提供冲突解决建议
- **检测维度**:
  - 触发词精确匹配
  - 触发词语义相似度（阈值>0.7为高风险）
  - 功能描述相似度（阈值>0.6为相似功能）
- **核心脚本**: `scripts/detect_conflicts.py`

---

### 3. validator-audit（验证审核技能）⭐ 本Agent专用
- **路径**: `workspace-validator/skills/validator-audit/`
- **触发词**: "验证审核"、"检查技能规范"、"审计GitHub"、"隐私安全检查"、"合规检查"
- **功能**:
  - **技能规范验证**: SKILL.md格式、触发词、相对路径、配置分离
  - **隐私安全审计**: 密码、API Key、手机号、邮箱、身份证号扫描
  - **GitHub合规检查**: .gitignore、node_modules、敏感文件、目录结构
- **检查清单**:
  - [ ] SKILL.md 是否存在
  - [ ] 触发词是否明确
  - [ ] 描述是否清晰
  - [ ] 文件路径是否规范（相对路径）
  - [ ] 配置是否分离（config.json）
  - [ ] 无硬编码密码
  - [ ] 无 API Key/Token
  - [ ] 无手机号（脱敏）
  - [ ] 无邮箱地址（脱敏）
  - [ ] 无个人身份信息

## 📋 验证审核Agent 工作流建议

### 日常审计流程

```
1. 技能规范检查
   └─ 使用 skill-conflict-detector 扫描技能冲突
   └─ 手动抽查 SKILL.md 格式规范

2. 隐私安全检查
   └─ 使用 github-compliance-checker 扫描敏感信息
   └─ 检查新增文件是否含硬编码密码/API Key

3. GitHub 合规检查
   └─ 检查 .gitignore 配置
   └─ 确认 logs/、memory/ 等隐私目录被正确忽略

4. 测试验证
   └─ 使用 auto-tester 执行回归测试
   └─ 验证核心技能功能正常
```

### 定期审计任务

| 频率 | 任务 | 使用技能 |
|------|------|----------|
| 每日 | 检查 GitHub 合规性 | github-compliance-checker |
| 每周 | 技能冲突检测 | skill-conflict-detector |
| 每周 | 代码统计 | code-stats |
| 每月 | 全面技能审计 | validator-audit |
| 发布前 | 回归测试 | auto-tester |
