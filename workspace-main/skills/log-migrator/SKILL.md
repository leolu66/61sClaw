工作日志归档和快速备份工具，支持两种独立功能。

## 功能 1：日志归档

### 功能描述

将工作区 `logs` 目录下的所有日志文件复制到外部归档目录，并在归档目录下建立当前日期的子目录。

### 目录结构

```
源目录（工作区）:
workspace-main/logs/
├── daily/           # 每日会话日志
├── tasks/           # 任务执行日志
├── errors/          # 错误日志
└── summary/         # 阶段总结

目标目录（归档）:
D:\openclaw\archive\
└── 2026-03-03/      # 按日期创建的子目录
    ├── daily/
    ├── tasks/
    ├── errors/
    └── summary/
```

### 输入/输出

- **输入**: 无（自动运行）
- **输出**:
  - 归档文件列表
  - 归档统计信息（文件数量、总大小）
  - 归档目标路径

### 配置项

在 `scripts/config.json` 中可调整以下参数：

```json
{
  "log_archive": {
    "source_dir": "logs",
    "archive_base": "D:\\\\openclaw\\\\archive",
    "create_date_subdir": true
  }
}
```

| 参数 | 说明 | 默认值 |
|------|------|--------|
| source_dir | 工作区日志源目录（相对路径） | logs |
| archive_base | 归档根目录（绝对路径） | D:\openclaw\archive |
| create_date_subdir | 是否创建日期子目录 | true |

### 输出示例

```
归档目录: D:\openclaw\archive\2026-03-03
```

## 功能 2：快速备份

将当前

### 功能描述工作区完整备份到指定位置。具体步骤：

1. 找到目标目录 `D:\anthropic\.openclaw\workspace-main`
2. 将已有的 `workspace-main` 重命名为带日期后缀的备份（如 `workspace-main.260303bak`）
3. 将当前工作区的 `workspace-main` 所有内容复制到目标位置

### 目录结构

```
备份前:
D:\anthropic\.openclaw\
└── workspace-main/          # 旧备份（将被重命名）

备份操作:
1. 重命名: workspace-main → workspace-main.260303bak
2. 复制: 当前 workspace-main → D:\anthropic\.openclaw\workspace-main

备份后:
D:\anthropic\.openclaw\
├── workspace-main/           # 新备份（刚复制）
└── workspace-main.260303bak/ # 旧备份（重命名后）
```

### 输入/输出

- **输入**: 无（自动运行）
- **输出**:
  - 备份操作步骤
  - 备份文件统计（文件数量、总大小）
  - 备份目标路径

### 配置项

在 `scripts/config.json` 中可调整以下参数：

```json
{
  "quick_backup": {
    "target_base": "D:\\\\anthropic\\\\.openclaw",
    "backup_suffix": "bak",
    "date_format": "%y%m%d"
  }
}
```

| 参数 | 说明 | 默认值 |
|------|------|--------|
| target_base | 备份目标根目录 | D:\anthropic\.openclaw |
| backup_suffix | 备份后缀 | bak |
| date_format | 日期格式（用于后缀） | %y%m%d |

### 输出示例

```
============================================================
快速备份
============================================================
当前工作区: C:\Users\luzhe\.openclaw\workspace-main
备份目标: D:\anthropic\.openclaw\workspace-main
------------------------------------------------------------
📋 检查目标目录...

  ✅ 目标目录存在: D:\anthropic\.openclaw\workspace-main
  🔄 将重命名为: workspace-main.260303bak

------------------------------------------------------------
📦 开始备份...

  ✅ 重命名旧备份: workspace-main → workspace-main.260303bak
  ✅ 复制文件到目标目录...
  ✅ 复制完成

------------------------------------------------------------
✅ 备份完成！
  备份文件: 128
  总大小: 3.2 MB
  备份目录: D:\anthropic\.openclaw\workspace-main

============================================================
```

## 功能 3：清理日志

### 功能描述

先归档日志，然后删除 workspace 下 logs 目录中所有超过指定天数的文件。

### 执行步骤

1. **步骤1**：归档日志（调用日志归档功能）
2. **步骤2**：清理超过天数的日志文件（默认 3 天）

### 输入/输出

- **输入**: 无（自动运行）
- **输出**:
  - 归档文件统计（文件数量、总大小）
  - 删除文件统计（文件数量、释放空间）

### 配置项

在 `scripts/config.json` 中可调整以下参数：

```json
{
  "log_cleanup": {
    "source_dir": "logs",
    "days_to_keep": 3,
    "dry_run": false
  }
}
```

| 参数 | 说明 | 默认值 |
|------|------|--------|
| source_dir | 日志源目录（相对路径） | logs |
| days_to_keep | 保留天数（超过此天数的文件将被删除） | 3 |
| dry_run | 试运行模式，只显示不删除 | false |

### 输出示例

```
============================================================
清理日志
============================================================
工作区: C:\Users\luzhe\.openclaw\workspace-main
------------------------------------------------------------

[STEP 1] 归档日志...

[OK] 归档完成！
  归档文件: 2
  总大小: 15.2 KB
  归档目录: D:\openclaw\archive\2026-03-03

[STEP 2] 清理旧日志...

------------------------------------------------------------
[INFO] 清理 3 天前的日志文件...
[INFO] 截止日期: 2026-03-01 00:00:00
------------------------------------------------------------
  [DEL] daily/2026-02-21-001-会话总结.md (年龄: 10 天)
  [DEL] daily/2026-02-27-003-会话总结.md (年龄: 4 天)

------------------------------------------------------------
```

## 边界情况

### 日志归档
- **源目录不存在**：提示错误，不执行归档
- **目标目录不存在**：自动创建
- **文件已存在**：覆盖（因为是同名归档）
- **无文件需要归档**：提示"没有文件需要归档"

### 快速备份
- **目标目录不存在**：提示错误，不执行备份
- **旧备份重命名失败**（已存在同名备份）：提示用户手动处理
- **复制失败**：保留已重命名的旧备份，报告错误
- **当前工作区为空**：提示警告但继续执行

### 清理日志
- **源目录不存在**：提示错误，不执行清理
- **dry_run 为 true**：只显示要删除的文件，不实际删除
- **无文件需要清理**：提示"没有文件需要清理"
- **归档失败**：不执行清理步骤，直接报错

---

## 注意事项

1. **日志归档是复制，不是移动**：原始日志保留在原位置
2. **快速备份会重命名旧备份**：确保有足够磁盘空间
3. **备份是完整复制**：包括所有子目录和隐藏文件
4. **建议定期手动备份**：不要依赖单一备份
