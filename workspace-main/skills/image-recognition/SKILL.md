# 图像识别技能

## 需求说明（SRS）

### 触发条件
用户上传图片并附带以下指令时触发：
- "识别这张图片"
- "分析图片内容"
- "提取图中文字"
- "识别文档截图"
- "分析PPT框架"
- "图片里有什么"
- "解读这张图片"

### 功能描述
用户上传图片后，**主 Agent 直接使用视觉能力分析图片**，无需创建子代理。支持多种识别场景：

1. **文档截图 OCR** - 提取图片中的文字内容，保持排版格式
2. **图片物体识别** - 识别图片中的物体、场景、人物等
3. **PPT 框架图识别** - 分析 PPT 截图的结构、流程、层级关系

### 输入/输出
- **输入**: 
  - 图片文件（用户上传）
  - 识别场景类型（根据指令自动判断：ocr/object/ppt）
- **输出**: 
  - 结构化的图像分析结果
  - 文字内容（OCR 场景）
  - 物体/场景描述（识别场景）
  - 框架结构分析（PPT 场景）

### 依赖条件
- 主 Agent 使用支持视觉的模型（当前: whalecloud/kimi-k2.5）
- 用户需要提供图片文件路径或上传图片

### 边界情况
- 图片格式不支持时提示用户
- 图片过大时建议压缩
- 识别失败时返回错误信息
- 无法判断场景时默认使用通用描述

## 使用方法

### 基本用法

用户直接上传图片并说明需求：

```
[上传图片]
识别这张文档截图中的文字
```

```
[上传图片]
分析这张PPT的框架结构
```

```
[上传图片]
图片里有什么物体？
```

### 场景说明

系统会自动根据用户指令判断识别场景：

| 场景 | 关键词 | 功能 |
|------|--------|------|
| OCR | 文字、提取、OCR、文档、截图 | 提取图片中的文字 |
| 物体识别 | 有什么、识别、物体、内容、描述 | 描述图片中的物体和场景 |
| PPT分析 | PPT、框架、结构、流程图、架构 | 分析框架结构和层级关系 |

## 实现原理

### 架构流程（简化版）

```
用户上传图片 + 指令
    ↓
主 Agent 接收图片和指令
    ↓
判断识别场景（OCR/物体/PPT）
    ↓
主 Agent 直接使用视觉能力分析
    ↓
返回结构化结果
```

**特点**：无需创建子代理，主 Agent 直接处理，响应更快

### 场景提示词

#### OCR 场景
```
请提取图片中的所有文字内容，注意：
1. 保持原有的排版格式，尽量还原段落和换行
2. 识别表格结构（如有），用表格形式输出
3. 区分标题和正文，标题可以加粗
4. 如有代码，保持代码格式
5. 输出 Markdown 格式
```

#### 物体识别场景
```
请详细描述图片内容，包括：
1. 图片中的主要物体和元素
2. 场景环境（室内/室外、自然/人工等）
3. 人物（如有）的数量、动作、表情、穿着
4. 颜色、风格、光线等视觉特征
5. 图片的整体氛围和主题
```

#### PPT 框架分析场景
```
请分析这张 PPT 或框架图的结构：
1. 整体框架层级和组成部分
2. 各部分之间的关系（并列、层级、流程等）
3. 核心观点、结论或关键信息
4. 逻辑流程（如有箭头或步骤）
5. 用层级列表或流程图形式输出结构
```

## 注意事项

- 图片大小建议不超过 10MB
- 支持的格式：JPG、PNG、GIF、WebP
- 复杂图片可能需要更长的处理时间
- 识别结果仅供参考，重要内容请人工核对
- **主 Agent 模型必须支持视觉能力**（当前: whalecloud/kimi-k2.5）
